    • 任何时刻指针i左边的数一定 ≤ x，指针j右边的数一定 ≥ x

    • 为什么初始化 i = l - 1, j = r + 1?

      先不管三七二十一，让i往右边移动，让j往左边移动

    • 为什么swap()前要判断i < j?

      i与j没有相遇才能交换

在第④步，`i == j` 指针相遇结束划分，不能进行交换

  • 为什么quick_sort(q, l, j)?可不可以变为quick_sort(q, l, i)?

    结束一次划分后，i == j,为了完成排序必须递归左半部分和右半部分。

    可以变成quick_sort(q, l, i) 。但是要改变相应代码，否则出现死循环。

  • 当代码写成quick_sort(q, l, j)时，不能初始化x = r。

    例如：样例为q[1,2]。

    第一次划分完，i == j == 1,递归左半区间quick_sort(q, 0, 1) ,递归右半区间quick_sort(q, 2, 2)，右半区间可以忽略了。问题出在左半区间，左半区间进行划分之后，又需要递归左半区间递归左半区间quick_sort(q, 0, 1) ，区间长度没有缩短，出现死循环。

  • 快排应用：找一个数组中第K大的数
